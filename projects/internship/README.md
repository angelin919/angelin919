# Стажировка: тестирование веб-приложений

## 1. Форма обратной связи
**Объект тестирования:** модальное окно с формой (ФИО, e-mail, компания, категория обращения, файл, текст).

**Результаты:**
- Всего тест-кейсов: **24**
- Пройдено: **54%** (13)
- Не пройдено: **42%** (10)
- Не выполнено: **4%** (1)

**Ключевые дефекты:**
- BUG-001: кнопка закрытия окна не работает.
- BUG-004: форма не отправляется без файла (нарушение требования).
- BUG-005: отсутствует реальный POST-запрос, вместо него появляется alert.
- BUG-009: кнопка «Очистить» сбрасывает не все поля.
- BUG-011: ошибки 404 при загрузке шрифтов и source maps.

**Техники тест-дизайна:** классы эквивалентности, граничные значения, позитивные/негативные сценарии, проверка XSS.
[Посмотреть можно здесь>>](https://disk.yandex.ru/d/e3ICWgGSRYV2Ww)

## 2. Комплексное тестирование (11 сайтов)
В ходе стажировки проведено функциональное тестирование 11 веб-сайтов (e‑commerce, сервисы). Основной фокус: навигация, поиск, авторизация.

### Пример: интернет-магазин нижнего белья
- **Проверено:** работа главного меню, выпадающие списки (hover), поведение шапки при скролле, иконки «Избранное» и «Корзина».
- **Найдено:** иконка «Избранное» ведёт в корзину (BR-001), ошибки в консоли при загрузке.

### Пример: поисковая система магазина
- **Сценарии:** полное/частичное название, опечатки, пустой запрос, синонимы.
- **Дефекты:** отсутствие обработки частичных запросов (BR-002), некорректная работа синонимов (BR-003, BR-004).

**Всего по направлению:** составлено **50+ тест-кейсов**, выявлено **15+ дефектов**, из них 3 критических (некорректная маршрутизация, потеря данных формы, отсутствие валидации).

**Применённые техники:** тест-дизайн (граничные значения, таблицы решений, попарное тестирование), тест-кейсы, регрессионное тестирование.
